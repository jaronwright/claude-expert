# Claude Expert — project notes

A Progressive Web App that lets signed-in users chat with an expert on Claude,
the Anthropic API, and Anthropic's research and products. Built as a portfolio
piece — opinionated, small surface area, deployable to Vercel in an afternoon.

---

## 1. Overview & architecture decisions

**Stack**

- **Next.js 14 App Router** for the whole app (pages + API routes in one process).
- **TypeScript strict** (`noUncheckedIndexedAccess`, `noImplicitOverride` on).
- **Tailwind CSS** with a small design-token set in `globals.css`.
- **Prisma + SQLite** locally. Schema is Postgres-compatible (one-line provider
  change to migrate).
- **NextAuth.js** with the GitHub OAuth provider and the Prisma adapter. Session
  strategy is `database`, so signing out genuinely invalidates server-side.
- **Anthropic AI SDK** (`ai` + `@ai-sdk/anthropic`) for streaming. Messages
  render progressively via the AI SDK's `useChat` hook.
- **next-pwa** wraps the app — auto-generates a service worker, precaches static
  assets, and serves a manifest. Disabled in dev so HMR isn't fighting the SW.

**Why these choices**

- **Server components + streaming API route.** The sidebar/list pages are RSC
  (fresh DB reads, no client fetch); the chat surface is a single client
  component driven by `useChat`. Simple mental model: server renders shell, one
  client island handles the live session.
- **Per-user encrypted API key + app-level fallback.** Realistic for a product
  (users bring their own key, admin can preload one for demos), and it exercises
  `crypto.createCipheriv` correctly with AES-256-GCM.
- **The conversation is created synchronously in the POST handler before
  streaming begins.** This is the cleanest way to return an `x-conversation-id`
  header that the client uses to rewrite the URL from `/chat` to `/chat/:id`.
  Messages are persisted in `onFinish` so partial generations during a network
  drop still leave a consistent DB state (just the user turn is saved).
- **System prompt lives in `lib/system-prompt.ts`.** It's long and opinionated —
  treat it as source code, not configuration. It references real
  `docs.anthropic.com` URLs; update them when Anthropic reorganises docs.

**File layout** — canonical Next.js 14 App Router.

```
app/
  (auth)/login/              sign-in surface
  (app)/                     authenticated area (sidebar layout)
    chat/page.tsx            new chat
    chat/[id]/page.tsx       existing conversation
    settings/                API key management
  api/
    auth/[...nextauth]/      NextAuth handler
    chat/route.ts            streaming chat endpoint
    conversations/           list + per-id GET/DELETE
    settings/api-key/        POST + DELETE for the encrypted key
components/
  chat/                      ChatWindow, MessageBubble, ChatInput, SuggestedQuestions, CodeBlock, Markdown
  sidebar/                   Sidebar, ConversationList
  ui/                        shadcn-style button / input / dialog
lib/
  anthropic.ts               API-key resolution + AI SDK client factory
  auth.ts                    NextAuth options
  db.ts                      Prisma singleton
  encryption.ts              AES-256-GCM helpers
  models.ts                  Allow-list of model IDs
  system-prompt.ts           Claude Expert persona
  utils.ts                   `cn` and date helpers
prisma/schema.prisma         DB schema
public/manifest.json         PWA manifest
public/icons/                Brand icons (generated by scripts/generate-icons.mjs)
```

## 2. Local run

Prerequisites: Node 20+ and a GitHub OAuth app you own.

```bash
# 1. Install dependencies (postinstall runs `prisma generate`).
npm install

# 2. Create your env file.
cp .env.example .env
# Then fill in the values — at minimum:
#   NEXTAUTH_SECRET   (openssl rand -base64 32)
#   ENCRYPTION_KEY    (openssl rand -base64 32)
#   GITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET
#   ANTHROPIC_API_KEY (optional — users can BYO in settings)

# 3. Create the SQLite database.
npx prisma db push

# 4. Start the dev server.
npm run dev
```

Sign in at http://localhost:3000/login. The first sign-in creates a `User` row
via the NextAuth Prisma adapter. Paste your Anthropic key in `/settings` (it's
encrypted at rest) or rely on the app-level `ANTHROPIC_API_KEY` fallback.

## 4. TODOs before going live

- **Postgres instead of SQLite.** In `prisma/schema.prisma`, change
  `provider = "sqlite"` to `provider = "postgresql"`. Provision Vercel Postgres
  (or Neon / Supabase), set `DATABASE_URL` (+ `DIRECT_URL` for connection
  pooling), and add `prisma migrate deploy` to the Vercel build command.
- **Real GitHub OAuth app.** Register one at
  https://github.com/settings/developers with callback
  `https://<your-domain>/api/auth/callback/github`.
- **Domain + `NEXTAUTH_URL`.** Set it to the production URL. Without this,
  OAuth callbacks will redirect to localhost.
- **Rotate `ENCRYPTION_KEY`.** Generate a fresh 32-byte base64 key for
  production. If you ever rotate it, saved user keys become unreadable —
  plan a rekey migration (decrypt with old, encrypt with new).
- **Rate limiting on `/api/chat`.** At minimum per-user; consider per-IP for
  abuse prevention. Wire up `@upstash/ratelimit` or Vercel's built-in limiter.
- **Analytics.** Vercel Analytics (first-party) + Anthropic usage dashboard is
  enough for v1. Add product events (chat sent, key saved) with PostHog if
  you need funnels.
- **Error reporting.** Sentry on both client and server — the streaming API
  route is the most important surface to monitor.
- **CSP + security headers.** Add `next.config.mjs` headers for CSP, HSTS,
  X-Frame-Options. Be careful with CSP and streaming responses.

## 5. Future ideas

- **Search over history.** Full-text across all conversations (SQLite FTS or
  Postgres `tsvector`), exposed as Cmd-K.
- **Export.** Download a conversation as Markdown or JSON.
- **Model selector polish.** Remember the last chosen model per user; show
  cost-per-conversation estimates.
- **Shareable read-only links.** Public URL that renders a saved conversation.
- **Usage stats.** Count tokens from the streamed response (`result.usage`)
  and show a lifetime + per-conversation breakdown in settings.
- **Prompt caching demo.** Add a one-click "cache the system prompt" toggle
  and display cache-hit % per request — good teaching aid for a portfolio piece.
- **Tool use playground.** Let the user define a mock tool inline and watch
  Claude call it. Fits the "expert on the API" theme well.
- **Multiplayer.** Shared conversations for teams. Invites via email.
- **Custom personas.** Let users write their own system prompts.
